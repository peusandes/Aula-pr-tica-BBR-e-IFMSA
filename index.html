Introdução à Meta-Análise

Aprenda na prática as etapas essenciais de uma revisão sistemática com meta-análise.

Como vai funcionar

Nesta aula prática, você vai passar por 3 etapas fundamentais de uma meta-análise:

  • 1
    Screening

    Você aprenderá a triar artigos usando o Rayyan, uma ferramenta online de screening colaborativo.

  • 2
    Extração de Dados

    Você vai extrair dados de um artigo real, preenchendo uma tabela de extração padronizada.

  • 3
    Análise Estatística

    Usando o Review Manager (RevMan), você vai criar um Forest Plot com os dados extraídos.

Navegue pelas etapas usando o menu lateral. Siga cada etapa na ordem para a melhor experiência!

Etapa 1

Screening de Artigos

Aprenda a selecionar estudos relevantes para sua revisão sistemática.

Acessar Rayyan Baixar arquivo .RIS

O que é o Screening?

O screening é a etapa de triagem onde avaliamos títulos e abstracts dos artigos encontrados na busca para decidir quais serão incluídos ou excluídos da revisão.

É uma das etapas mais importantes da revisão sistemática, pois garante que apenas estudos relevantes avancem para a análise.

Princípios de um bom screening

Use a pergunta PICO como guia:

  • P
    População — Qual a população do estudo?
  • I
    Intervenção — Qual intervenção está sendo avaliada?
  • C
    Comparação — Com o que está sendo comparada?
  • O
    Outcome / Desfecho — Quais são os desfechos medidos?
  • Sempre volte à sua pergunta de pesquisa antes de incluir ou excluir um artigo.
  • Na dúvida, inclua o artigo — é melhor avaliar o texto completo depois do que perder um estudo relevante.
  • O screening deve ser feito de forma independente por pelo menos 2 revisores.
  • Discordâncias devem ser resolvidas por consenso ou por um terceiro revisor.

Passo a passo no Rayyan

  1. Crie sua conta no Rayyan acessando o link acima.
  2. Crie um novo projeto (New Review) — dê um nome ao seu projeto.
  3. Importe o arquivo .RIS — baixe o arquivo usando o botão acima e faça upload no Rayyan.
  4. Desduplicação — antes de iniciar o screening, faça a desduplicação dos artigos. Isso pode ser feito diretamente no Rayyan (que detecta duplicatas automaticamente) ou previamente no Zotero (um gerenciador de referências gratuito).
  5. Convide colaboradores — adicione outros revisores ao projeto para o screening independente.
  6. Inicie o screening — leia título e abstract de cada artigo e classifique como:
    • ✅ Include — o estudo parece relevante para a pergunta PICO.
    • ❌ Exclude — o estudo claramente não atende aos critérios.
    • ❓ Maybe — na dúvida, marque como "maybe" para revisão posterior.
  7. Resolva conflitos — após todos os revisores finalizarem, resolva as discordâncias.

Dicas importantes

  • Use rótulos (labels) no Rayyan para marcar os motivos de exclusão.
  • Filtre por palavras-chave para agilizar o processo.
  • O Rayyan possui um sistema de inteligência artificial que sugere inclusões/exclusões — use como apoio, nunca como decisão final.
Etapa 2

Extração de Dados

Aprenda a extrair informações-chave dos estudos incluídos.

Baixar artigo (Singla 2015) Baixar Tabela de Extração

O que é a Extração de Dados?

Após o screening, os artigos incluídos passam pela extração de dados — o processo de coletar sistematicamente as informações relevantes de cada estudo.

Usamos uma tabela de extração padronizada para garantir que todos os revisores extraiam as mesmas informações de forma consistente.

Extração das Características Basais (Tabela 1)

A Tabela 1 de um artigo científico geralmente contém as baseline characteristics (características basais) dos participantes do estudo. Nela você encontrará informações como:

  • Nome do autor e ano de publicação
  • Tipo de estudo (RCT, coorte, caso-controle, etc.)
  • Número total de participantes (N)
  • Idade média (mean age) dos participantes
  • Distribuição por sexo
  • Outras características demográficas e clínicas relevantes

Como encontrar: Procure a seção "Results" do artigo e localize a "Table 1" ou "Baseline characteristics". Depois, preencha a tabela de extração com essas informações.

Extração dos Desfechos (Outcomes)

Além das características basais, precisamos extrair os dados dos desfechos (outcomes) para a análise estatística.

No nosso exemplo prático, vamos usar o estudo Singla et al. (2015), que avalia o manejo de aneurismas infecciosos intracranianos.

No exercício de hoje:
  • Nosso desfecho de interesse é a mortalidade.
  • O estudo compara:
    • Grupo exposição (tratamento cirúrgico — clipping/embolization): pacientes submetidos a clipping ou embolização.
    • Grupo controle (tratamento conservador — no treatment): pacientes que não receberam tratamento neurocirúrgico.
  • O que extrair:
    • N do evento (número de mortes) em cada grupo
    • N total (número total de pacientes) em cada grupo

Esses dados (eventos e totais por grupo) serão usados na próxima etapa para construir o Forest Plot no Review Manager.

  • Leia atentamente a seção de Resultados e as Tabelas do artigo.
  • Procure especificamente por dados de mortality comparando os grupos.
  • Anote de onde você extraiu cada dado para garantir transparência e reprodutibilidade.
Etapa 3

Análise Estatística — Forest Plot

Use o Review Manager para criar seu primeiro Forest Plot.

Baixar RevMan

O que é um Forest Plot?

  • O Forest Plot é a representação gráfica clássica de uma meta-análise.
  • Ele mostra visualmente o efeito de cada estudo individual e o efeito combinado (pooled effect).
  • Cada estudo é representado por um quadrado (cujo tamanho reflete o peso do estudo) e uma linha horizontal (intervalo de confiança).
  • O losango na parte inferior representa o efeito combinado de todos os estudos.

O que vamos fazer agora

  • Vamos usar o Review Manager (RevMan) — o software oficial da Cochrane para revisões sistemáticas.
  • Utilizaremos os dados de mortalidade que você extraiu na etapa anterior (Singla 2015) para criar um Forest Plot.
  • Este é um exercício simplificado com apenas um estudo para que você entenda a mecânica — em uma meta-análise real, teríamos múltiplos estudos.

Passo a passo no RevMan

  1. Abra o RevMan que você baixou anteriormente.
  2. Crie um novo projeto de revisão.
  3. Adicione um desfecho — selecione dichotomous outcome, pois estamos analisando mortalidade (evento: sim/não).
  4. Insira os dados que você extraiu:
    • Nome do estudo: Singla 2015
    • Grupo experimental (Clipping/Embolization): N do evento e N total
    • Grupo controle (No treatment): N do evento e N total
  5. Gere o Forest Plot — o RevMan calculará automaticamente o Risk Ratio (ou Odds Ratio), o intervalo de confiança e gerará o gráfico.
  6. Interprete o resultado:
    • Se o efeito estiver à esquerda da linha de nulidade (1 para RR/OR) → favorece o grupo experimental.
    • Se estiver à direita → favorece o grupo controle.
    • Se o intervalo de confiança cruzar a linha de nulidade → resultado não é estatisticamente significativo.

Conceitos-chave

  • Heterogeneidade (I²)

    Mede a variabilidade entre os estudos. Com apenas um estudo, não se aplica.

  • Peso do estudo

    Em meta-análises com múltiplos estudos, cada estudo contribui proporcionalmente ao seu tamanho amostral e precisão.

  • Efeito fixo vs. efeito aleatório

    Dois modelos estatísticos para combinar resultados — o modelo de efeito aleatório é mais conservador e geralmente preferido quando há heterogeneidade.
